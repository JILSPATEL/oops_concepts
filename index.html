Class & Object
Blueprint and its instances
▼
Class = a blueprint/template that defines attributes (data) and methods (behavior). Object = a real-world instance created from that class.

Key Points

A class is like a cookie-cutter; objects are the cookies
Class is declared once; objects can be created many times
Each object has its own copy of member variables (non-static)
Objects communicate via method calls
C++class Car {
public:
    // Member variables (attributes)
    string brand;
    int speed;

    // Member function (method)
    void drive() {
        cout << brand << " is driving at " << speed << " km/h";
    }
};

int main() {
    Car myCar;          // Object created on stack
    myCar.brand = "Toyota";
    myCar.speed = 100;
    myCar.drive();       // Output: Toyota is driving at 100 km/h

    Car* carPtr = new Car(); // Object on heap
    carPtr->brand = "Honda";
    delete carPtr;       // Must free heap memory!
}
Javaclass Car {
    // Member variables (attributes)
    String brand;
    int speed;

    // Member method
    void drive() {
        System.out.println(brand + " is driving at " + speed + " km/h");
    }
}

class Main {
    public static void main(String[] args) {
        Car myCar = new Car(); // All Java objects are on heap
        myCar.brand = "Toyota";
        myCar.speed = 100;
        myCar.drive();          // Output: Toyota is driving at 100 km/h
        // No delete needed — Garbage Collector handles memory
    }
}

Constructor & Destructor
Special methods for object creation and cleanup
▼
Constructor = auto-called when object is created. Used to initialize values. Destructor = auto-called when object is destroyed. Used to free resources. (Java has no destructor — uses Garbage Collector)

Key Points

Constructor has same name as class, no return type
Can be overloaded (multiple constructors with different params)
Default constructor = no parameters. Parameterized = takes arguments
Copy constructor (C++) = creates object from another object
In Java, destructor equivalent is finalize() — but rarely used
C++class Student {
public:
    string name;
    int age;

    // Default constructor
    Student() {
        name = "Unknown"; age = 0;
        cout << "Object Created!\n";
    }

    // Parameterized constructor
    Student(string n, int a) : name(n), age(a) {
        cout << name << " created!\n";
    }

    // Copy constructor
    Student(const Student& s) {
        name = s.name; age = s.age;
        cout << "Copied!\n";
    }

    // Destructor (~ prefix)
    ~Student() {
        cout << name << " destroyed!\n";
    }
};

int main() {
    Student s1;               // Default constructor
    Student s2("Alice", 20); // Parameterized
    Student s3 = s2;          // Copy constructor
} // Destructors called automatically here
Javaclass Student {
    String name;
    int age;

    // Default constructor
    Student() {
        name = "Unknown"; age = 0;
        System.out.println("Object Created!");
    }

    // Parameterized constructor
    Student(String n, int a) {
        this.name = n;   // 'this' refers to current object
        this.age = a;
        System.out.println(name + " created!");
    }

    // No destructor in Java — GC handles cleanup
    // finalize() is deprecated in newer Java versions
}

class Main {
    public static void main(String[] args) {
        Student s1 = new Student();           // Default
        Student s2 = new Student("Alice", 20); // Parameterized
    }
}

C++class BankAccount {

    string pin;

1. Encapsulation
Hiding internal data, exposing only what's needed
▼
Encapsulation = wrapping data (variables) and methods together into a single unit (class), and restricting direct access to internal data using private variables + public getters/setters.

Why use it?

Data protection — prevents unauthorized modification
Flexibility — can change internal implementation without affecting users
Validation — setters can check values before assigning
Real-life analogy: ATM machine — you interact via buttons, not internal circuits
C++class Person {
private:
    string name;
    int age;     // Hidden from outside

public:
    // Setter with validation
    void setAge(int a) {
        if(a > 0 && a < 150) age = a;
        else cout << "Invalid age!";
    }

    // Getter
    int getAge() { return age; }
    void setName(string n) { name = n; }
    string getName() { return name; }
};

int main() {
    Person p;
    p.setAge(25);     // ✓ Controlled access
    p.setAge(-5);    // ✗ Rejected by validation
    // p.age = -5;   ERROR! Private
}
Javaclass Person {
    private String name;
    private int age;    // Encapsulated - hidden!

    // Setter with validation
    public void setAge(int a) {
        if(a > 0 && a < 150) this.age = a;
        else System.out.println("Invalid age!");
    }

    // Getter
    public int getAge() { return age; }

    public void setName(String n) { this.name = n; }
    public String getName() { return name; }
}

class Main {
    public static void main(String[] args) {
        Person p = new Person();
        p.setAge(25);      // ✓ Valid
        p.setAge(-5);     // Rejected by setter
        // p.age = 25;    ERROR! private
    }
}

2. Inheritance
Child class acquires properties of parent class
▼
Inheritance = a class (child/derived) inherits properties and methods from another class (parent/base). Promotes code reuse and establishes an "IS-A" relationship.

Types of Inheritance

Single: A → B (one parent, one child)
Multilevel: A → B → C (chain)
Multiple: A, B → C (C++ supports, Java does NOT via classes — only via interfaces)
Hierarchical: A → B, A → C (one parent, multiple children)
Java uses extends; C++ uses : with access specifier
C++// Base (Parent) class
class Animal {
public:
    string name;
    void eat() { cout << name << " is eating\n"; }
    virtual void sound() { cout << "Some sound\n"; }
};

// Derived (Child) class — inherits from Animal
class Dog : public Animal {
public:
    void fetch() { cout << name << " fetches ball!\n"; }
    void sound() override { cout << "Woof!\n"; } // Override
};

// Multilevel inheritance
class GuideDog : public Dog {
public:
    void guide() { cout << "Guiding owner!\n"; }
};

int main() {
    Dog d;
    d.name = "Rex";
    d.eat();   // Inherited from Animal
    d.fetch(); // Own method
    d.sound(); // Overridden: "Woof!"
}
Java// Base (Parent) class
class Animal {
    String name;
    void eat() { System.out.println(name + " is eating"); }
    void sound() { System.out.println("Some sound"); }
}

// Child class — use 'extends' keyword
class Dog extends Animal {
    void fetch() { System.out.println(name + " fetches ball!"); }

    @Override
    void sound() { System.out.println("Woof!"); } // Override
}

// Multilevel
class GuideDog extends Dog {
    void guide() { System.out.println("Guiding owner!"); }
}

class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.name = "Rex";
        d.eat();    // Inherited
        d.fetch();  // Own method
        d.sound();  // Overridden: "Woof!"

        // super keyword — access parent's method
        // super.sound() would call Animal's sound()
    }
}

3. Polymorphism
One interface, many forms
▼
Polymorphism = "many forms". Same function name behaves differently based on context. Two types: Compile-time (method overloading) and Runtime (method overriding via virtual functions).

Overloading vs Overriding

Overloading (Compile-time): Same name, different parameters — resolved at compile time
Overriding (Runtime): Child redefines parent method — resolved at runtime via vtable
In C++, use virtual keyword for runtime polymorphism
In Java, all non-static methods are virtual by default
C++// COMPILE-TIME POLYMORPHISM (Overloading)
class Calculator {
public:
    int add(int a, int b) { return a+b; }
    double add(double a, double b) { return a+b; } // Same name!
    int add(int a, int b, int c) { return a+b+c; }
};

// RUNTIME POLYMORPHISM (Overriding + virtual)
class Shape {
public:
    virtual double area() { return 0; } // virtual!
    virtual ~Shape() {}
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() override { return 3.14 * r * r; }
};

class Rect : public Shape {
    double w, h;
public:
    Rect(double w, double h) : w(w), h(h) {}
    double area() override { return w * h; }
};

int main() {
    Shape* s1 = new Circle(5);
    Shape* s2 = new Rect(4, 6);

    cout << s1->area(); // Calls Circle::area() — 78.5
    cout << s2->area(); // Calls Rect::area()   — 24
    // Same pointer type (Shape*), different behavior!
}
Java// COMPILE-TIME: Method Overloading
class Calculator {
    int add(int a, int b) { return a+b; }
    double add(double a, double b) { return a+b; }
    int add(int a, int b, int c) { return a+b+c; }
}

// RUNTIME: Method Overriding
class Shape {
    double area() { return 0; } // All methods virtual in Java
}

class Circle extends Shape {
    double r;
    Circle(double r) { this.r = r; }

    @Override
    double area() { return 3.14 * r * r; }
}

class Rect extends Shape {
    double w, h;
    Rect(double w, double h) { this.w=w; this.h=h; }

    @Override
    double area() { return w * h; }
}

class Main {
    public static void main(String[] args) {
        Shape s1 = new Circle(5);  // Upcasting
        Shape s2 = new Rect(4, 6);

        System.out.println(s1.area()); // 78.5 — Circle's area
        System.out.println(s2.area()); // 24   — Rect's area
    }
}

4. Abstraction
Hiding complexity, showing only essentials
▼
Abstraction = showing only essential features and hiding implementation details. Achieved via abstract classes and interfaces. Real-life analogy: You drive a car without knowing how the engine works.

Abstract Class vs Interface

Abstract class: Can have both abstract (no body) and concrete (with body) methods. Can have constructors and state
Interface: All methods are abstract by default (in Java 7). A class can implement multiple interfaces (solves multiple inheritance problem)
C++ uses pure virtual functions (= 0) for abstraction
You CANNOT create an object of an abstract class directly
C++// Abstract class (has at least one pure virtual function)
class Vehicle {
public:
    virtual void start() = 0;   // Pure virtual = abstract
    virtual void stop() = 0;    // Must be overridden

    void fuel() {                 // Concrete method
        cout << "Adding fuel...\n";
    }
    virtual ~Vehicle() {}
};

// Must implement ALL pure virtual methods
class Car : public Vehicle {
public:
    void start() override { cout << "Car started!\n"; }
    void stop()  override { cout << "Car stopped!\n"; }
};

int main() {
    // Vehicle v;  ERROR! Cannot instantiate abstract class
    Vehicle* v = new Car();
    v->start();   // Car started!
    v->fuel();    // Adding fuel...
    delete v;
}
Java// Abstract class
abstract class Vehicle {
    abstract void start();   // Abstract — no body
    abstract void stop();    // Must be implemented

    void fuel() {             // Concrete method
        System.out.println("Adding fuel...");
    }
}

// Interface — 100% abstract (Java 7 style)
interface Drivable {
    void drive();   // public abstract by default
    void park();
}

// Can extend ONE abstract class + implement MANY interfaces
class Car extends Vehicle implements Drivable {
    @Override public void start() { System.out.println("Car started!"); }
    @Override public void stop()  { System.out.println("Car stopped!"); }
    @Override public void drive() { System.out.println("Driving!"); }
    @Override public void park()  { System.out.println("Parked!"); }
}

class Main {
    public static void main(String[] args) {
        // Vehicle v = new Vehicle(); ERROR - abstract!
        Vehicle v = new Car();
        v.start();
        v.fuel();
    }
}

Friend Function (C++ only)
External function accessing private members
▼
A friend function is NOT a member of the class, but is granted access to its private and protected members. Declared using the friend keyword inside the class.
C++class Box {
private:
    double width;
public:
    Box(double w) : width(w) {}

    // Declare friend function
    friend double getWidth(Box b);
};

// NOT a member, but can access private 'width'
double getWidth(Box b) {
    return b.width; // Allowed because of friend!
}

int main() {
    Box b(10.5);
    cout << getWidth(b); // 10.5
}

Static Members
Shared across all objects of a class
▼
A static member (variable or method) belongs to the CLASS itself, not to any individual object. All objects share one copy of static variables.
C++class Counter {
public:
    static int count;  // Shared by ALL objects
    Counter() { count++; }
    static int getCount() { return count; }
};

int Counter::count = 0; // Must define outside class

int main() {
    Counter c1, c2, c3;
    cout << Counter::getCount(); // 3
}
Javaclass Counter {
    static int count = 0;  // Shared by ALL objects

    Counter() { count++; }

    static int getCount() { return count; }
}

class Main {
    public static void main(String[] args) {
        Counter c1 = new Counter();
        Counter c2 = new Counter();
        Counter c3 = new Counter();
        System.out.println(Counter.getCount()); // 3
    }
}
🔁
this Pointer / Keyword
Reference to current object
▼
this refers to the current object that called the method. Used to distinguish between member variables and parameters with the same name, and to return the current object.
C++class MyClass {
    int x;
public:
    void setX(int x) {
        this->x = x; // this->x = member, x = param
    }
    // Method chaining with 'this'
    MyClass& increment() {
        x++;
        return *this; // Return current object
    }
    void print() { cout << x; }
};

int main() {
    MyClass obj;
    obj.setX(5);
    obj.increment().increment().print(); // 7 (chaining!)
}
Javaclass MyClass {
    int x;

    void setX(int x) {
        this.x = x; // this.x = member, x = parameter
    }

    // Constructor chaining with this()
    MyClass() {
        this(0); // Calls parameterized constructor
    }
    MyClass(int x) {
        this.x = x;
    }

    // Return this for method chaining
    MyClass increment() {
        x++;
        return this;
    }
}

class Main {
    public static void main(String[] args) {
        MyClass obj = new MyClass(5);
        obj.increment().increment(); // x = 7
    }
}

⚡ Quick Interview Answers

The one-liners you need to know cold

What is OOP?

A programming paradigm based on the concept of objects that contain data and code, centered around 4 pillars: Encapsulation, Inheritance, Polymorphism, Abstraction.

Class vs Object

Class is the blueprint/template. Object is an instance. Class = cookie-cutter. Objects = cookies made from it.

Encapsulation

Bundling data + methods together and hiding internal details. Achieved via private fields + public getters/setters.

Inheritance

Child class inherits properties of parent. Promotes code reuse. "IS-A" relationship. C++: :, Java: extends.

Polymorphism

Same interface, different behavior. Overloading = compile-time. Overriding = runtime (needs virtual in C++).

Abstraction

Show only what's necessary, hide implementation. Done via abstract classes and interfaces.

Abstract class vs Interface

Abstract class: partial implementation, can have constructor. Interface: 100% abstract, a class can implement many.

Constructor

Special method called automatically when object is created. Same name as class, no return type. Can be overloaded.

C++ Destructor

~ClassName() — called when object is destroyed. Java has no destructor (GC handles it). Always make virtual in base class!

Virtual function (C++)

Enables runtime polymorphism. Without virtual, base class method is called even for derived objects. Java: all methods are virtual by default.

this pointer

Refers to the current object. Used to resolve naming conflicts between parameters and member variables.

static member

Belongs to the class, not individual objects. All instances share one copy. Access via ClassName::member (C++) or ClassName.member (Java).
